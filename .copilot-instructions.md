# Copilot Build Instructions for RasPin Pinball

## Build Methodology

**IMPORTANT**: For all Raspberry Pi/Debian builds, always use `run_in_terminal` with `isBackground=false` instead of `run_task`. This ensures compilation completes before analyzing output and prevents premature error checking.

## Project Structure

- **Source files**: `src/system/*.cpp`, `src/user/*.cpp`, `src/3rdparty/*.cpp`
- **Build output (Windows debug)**: `build/windows/debug/Pinball.exe`
- **Build output (Raspberry Pi debug)**: `build/raspi/debug/Pinball`
- **Build output (Raspberry Pi release)**: `build/raspi/release/Pinball`
- **Build output (Debian debug)**: `build/debian/debug/Pinball_DebianOS`
- **Build output (Debian release)**: `build/debian/release/Pinball_DebianOS`
- **Utilities**: `src/PButils/` (FontGen, pblistdevices, pbsetamp, pblaunch)
- **Include paths**: `src/include_ogl_raspi/` (Linux), `src/include_ogl_win/` (Windows)
- **Scripts**: `scripts/update_version.py`, `scripts/generate_io_header.py`

## Raspberry Pi Build Commands

### Full Raspberry Pi Debug Build
Use the VS Code task **"Raspberry Pi: Full Pinball Build"**, or manually:

```bash
cd <project_root> && python3 scripts/update_version.py && python3 scripts/generate_io_header.py && /usr/bin/g++ -g -DEXE_MODE_RASPI -o build/raspi/debug/Pinball \
  src/3rdparty/stb_image.cpp src/3rdparty/cgltf.cpp \
  src/system/PBOGLES.cpp src/system/PB3D.cpp src/system/PBGfx.cpp \
  src/system/PBLinuxRender.cpp src/system/PBDebounce.cpp \
  src/user/PBDevice.cpp src/system/Pinball_IO.cpp src/system/Pinball_Table.cpp \
  src/user/tablemodes/Pinball_Table_ModeInit.cpp \
  src/user/tablemodes/Pinball_Table_ModeStart.cpp \
  src/user/tablemodes/Pinball_Table_ModeMain.cpp \
  src/user/tablemodes/Pinball_Table_ModeReset.cpp \
  src/user/tablemodes/Pinball_Table_ModeGameEnd.cpp \
  src/user/tablemodes/Pinball_Table_ModePlayerEnd.cpp \
  src/system/PBSound.cpp src/system/PBVideo.cpp src/system/PBVideoPlayer.cpp \
  src/system/PinballMenus.cpp src/user/PBSequences.cpp \
  src/system/Pinball_Engine.cpp src/system/Pinball.cpp \
  -I src -I src/system -I src/user -I src/3rdparty -I src/include_ogl_raspi \
  -I/usr/include/EGL -I/usr/include/GLES3 -I/usr/include/KHR \
  -I/usr/include/X11 -I/usr/include/X11/extensions \
  -lEGL -lGLESv2 -lpthread -lm -lX11 -lXrandr -lwiringPi -lSDL2 -lSDL2_mixer \
  -lavcodec -lavformat -lavutil -lswscale -lswresample
```

### Debian Debug Build
Use the VS Code task **"DebianOS: Full Pinball Build"** or substitute `-DEXE_MODE_DEBIAN` and output to `build/debian/debug/Pinball_DebianOS` (omit `-lwiringPi`).

## Build Process Notes

1. **Always run version update first**: `python3 scripts/update_version.py`
2. **Always regenerate IO header**: `python3 scripts/generate_io_header.py`
3. **Check for new source files**: Before building, verify all `.cpp` files in `src/system/` and `src/user/` are included
4. **Wait for compilation to complete** before analyzing output
5. **Use absolute paths** in all build commands

### Source File Verification

Before each build, check if new `.cpp` files have been added:

```bash
find src/system src/user src/3rdparty -name "*.cpp" -type f | sort
```

## Preferred Tool Usage

- ✅ **USE**: `run_in_terminal(command, explanation, isBackground=false)`
- ❌ **AVOID**: `run_task()` for builds (returns before completion)

## Error Handling

When build errors occur:
1. Wait for complete compilation output
2. Identify specific error lines and files
3. Fix errors before attempting another build
4. Re-run full build to verify fixes

## Common Tasks

- **Clean Windows build**: delete `build/windows/debug/` or `build/windows/release/` contents
- **Clean Linux build**: delete `build/raspi/debug/Pinball` or `build/raspi/release/Pinball`
- **Version update**: always included in VS Code build task dependency chain
- **Verify source files**: `find src/system src/user -name "*.cpp" -type f | sort`
